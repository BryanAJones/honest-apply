# Freelance profile — EXAMPLE (a designer, to show the track is discipline-agnostic)

> A filled-in example of the Freelance Track sections (template §4-8) for **Maya, a freelance
> product/brand designer** — deliberately *not* a developer, to show the Proof Ledger and Profile
> Pack work for any discipline. This is illustrative; your real one comes from
> `job-application-init` (freelance track) + `job-application-profile`.

## §4 — Freelance Targeting Spec
- **Services:**
  - Brand identity (logo + system) — fixed from $2,500
  - Product UI / app screens — $90/hr
  - Design system / component library — project min $4,000
- **Target platforms:** Contra, Dribbble, Wellfound
- **Rate floor:** $70/hr
- **Bid budget:** n/a (chosen platforms are free to apply)
- **Cold-start:** `{ enabled: true, gigs_remaining: 4 }` (new to freelancing; underprice early
  to earn first reviews)

## §5 — Profile Pack (neutral source)
- **Positioning:** "Product & brand designer who ships polished UI and the system behind it."
- **Overview:** Lead line names the work + the proof (a shipped client app). Then range
  (brand → product → systems). Ends honest: newer to *freelance*, so offers a small paid first
  milestone.
- **Rate card:** published $90/hr · floor $70/hr · project min $2,500
- **Portfolio projects:** see Proof Ledger below (the presentable view).

## §6 — Proof Ledger (4 honesty fields per entry — the discipline-agnostic part)
- **Acme app redesign** (service: product UI)
  - Link/media: live app + Dribbble case study (public)
  - Solo/team: **team** — my slice: the onboarding + checkout screens (not the whole app)
  - Maturity/traction: **live-with-users**
  - Demonstrable: working-demo + design-screenshots
  - IP: public-ok (client approved the case study)
- **Bloom coffee brand identity** (service: brand identity)
  - Link/media: Behance + the live packaging (public)
  - Solo/team: **solo**
  - Maturity/traction: **shipped-at-scale** (on shelves)
  - Demonstrable: public-link + photos
  - IP: public-ok
- **"Tide" finance app concept** (service: product UI)
  - Link/media: Dribbble shots
  - Solo/team: **solo**
  - Maturity/traction: **prototype** (concept, never shipped — say so)
  - Demonstrable: design-screenshots-only
  - IP: public-ok

> Note how the honesty fields keep the concept piece from being dressed up as shipped work, and
> keep the team app from being claimed solo — exactly the guards a developer profile uses, with
> zero code involved.

## §7 — Proposal Spine
- Opener: restate the client's design problem in their words.
- Proof to attach: the ONE most relevant ledger entry (brand gig → Bloom; app gig → Acme).
- Paid-trial default: **yes** (new freelancer) — "let me do the first screen / one logo
  direction for $X so you can see the fit."

## §8 — Platform Plan (recommender output)
- **Design wedge:** START HERE → **Contra** + **Dribbble** (free, portfolio-forward; visual
  proof is the whole game for designers) · ADD → **Wellfound** (startups needing product
  design) · SKIP/LATER → **Upwork** until there are reviews.
  *(cost to apply: all free; facts verified 2026-06-03)*
